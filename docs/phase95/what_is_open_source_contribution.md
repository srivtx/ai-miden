# What is an Open Source Contribution?

## Problem it Solves
Reading about machine learning is not enough. Real skill comes from wrestling with production-grade code, tests, and documentation. Open source provides that battlefield.

## Definition
An open source contribution is a code, documentation, or design change submitted to a publicly maintained software project. In ML, this often means libraries like scikit-learn, NumPy, or JAX.

## Analogy
Contributing to open source is like apprenticing in a master craftsman's workshop. You learn by fixing real tools, reading expert code, and having your work reviewed by experienced practitioners.

## Example
A contributor notices that a scikit-learn classifier documentation lacks a note about memory usage. They open an issue, write a patch, add a test, and submit a pull request. The review process teaches them about API stability and backward compatibility.

## Common Confusion
Using an open source library is not contributing. Contribution requires giving back: reporting bugs, improving docs, or adding features that help the broader community.
