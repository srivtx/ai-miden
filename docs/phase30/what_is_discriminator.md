### 1. Why it exists (THE PROBLEM first)
If we only have a generator, we have no way to know whether its outputs are any good. Pixel-wise loss functions like MSE produce blurry results. We need a learned critic — a network that understands what "realistic" means for a given domain — to provide rich, high-level feedback to the generator.

### 2. Definition (very simple)
A Discriminator is a binary classifier network that takes an image (or any data sample) and outputs a probability that the sample is real (from the training dataset) rather than fake (produced by the generator). It is trained on a mix of real and generated samples.

### 3. Real-life analogy
An art inspector at a museum who specializes in spotting forgeries. The inspector studies thousands of real paintings to learn the subtle brush strokes, canvas textures, and pigment patterns that distinguish authentic art from fakes. When shown a new painting, the inspector gives a confidence score: "I am 95% sure this is real" or "This is definitely a forgery."

### 4. Tiny numeric example
Input image (2 pixels): x = [0.92, 0.15]

Discriminator network:
- Hidden layer: h = ReLU(W @ x + b) = [0.7, 0.3, 0.9]
- Output (probability real): p_real = sigmoid(v @ h + c) = 0.73

If x was from the real dataset, the target is 1.0. Loss = -log(0.73) = 0.31.
If x was generated by the generator, the target is 0.0. Loss = -log(1 - 0.73) = 1.31.

The discriminator gets better at recognizing fakes by minimizing these losses.

### 5. Common confusion
- **The discriminator is not a fixed judge.** It evolves alongside the generator. Early in training, the discriminator is easily fooled by crude fakes. Later, it becomes sophisticated, forcing the generator to improve.
- **Discriminator output is not ground truth.** A score of 0.99 does not mean the image is objectively perfect; it means the discriminator cannot tell it is fake. The discriminator might have blind spots.
- **The discriminator is usually discarded after training.** Once the generator is good enough, you only keep the generator. The discriminator has served its purpose as a training tool.
- **It can be used for feature extraction.** The hidden layers of a well-trained discriminator learn useful representations of the data, similar to a pre-trained classifier. Some methods use discriminator features for transfer learning.
- **Label smoothing helps.** Instead of targets being exactly 1 (real) and 0 (fake), using 0.9 and 0.1 prevents the discriminator from becoming overconfident too quickly.

### 6. Where it is used in our code
`src/phase30/phase30_gan.py` implements a toy discriminator that classifies 2D points as "real" (from a target distribution) or "fake" (from the generator), and both networks improve through adversarial training.
