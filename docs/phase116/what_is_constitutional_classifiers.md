## What Is Constitutional Classifiers?

---

### The Problem

A jailbreak exploit bypasses a model's safety training by wrapping a harmful request in a fictional narrative or a base64 encoding. Standard refusal mechanisms fail because the harmful content is obfuscated. The model sees the decoded intent but the safety layer does not. How do you build a defense that generalizes to novel obfuscation strategies without manually patching each one?

---

### Definition

**Constitutional Classifiers** are defense mechanisms that train a classifier on a set of constitutional principles (e.g., "do not assist with illegal acts") so it can detect jailbreaks, harmful inputs, and policy violations at inference time. The classifier operates on the input before the main model processes it, filtering adversarial examples that bypass the main model's own safety training.

**How it works:**
```
Constitutional principles → training data → classifier model
At inference time:
  1. User input arrives
  2. Constitutional classifier scores input for harmfulness / jailbreak likelihood
  3. If score exceeds threshold, input is blocked or routed to a safer fallback
  4. If score is low, input passes to the main model
Result: harmful inputs are caught before generation begins
```

**Key techniques:**
- Training on adversarial examples generated by automated red-teaming
- Constitutional principles as soft constraints in the classifier loss
- Ensemble of input-level and output-level classifiers
- Continuous updating with new jailbreak discoveries
- Confidence calibration to minimize false positives

**Why this matters:**
- Safety training inside the main model can be bypassed by sophisticated prompts
- A separate classifier adds a redundant safety layer
- Anthropic reported 3,000+ hours of red-teaming with no universal jailbreak when constitutional classifiers were active
- Input filtering is cheaper than output filtering because generation has not started yet

---

### Real-Life Analogy

A courthouse security checkpoint.
- **Main model safety training:** The judge inside the courtroom is trained to recognize disruptive behavior and eject offenders. But a clever litigant might disguise a threat as a legal argument, fooling the judge until it is too late.
- **Constitutional classifier:** Before entering the courthouse, everyone passes through a metal detector and a bag scanner. The scanners are trained on thousands of images of weapons concealed in everyday objects. They catch threats before the judge ever sees them. Even if the judge is distracted, the checkpoint provides a baseline of safety.
- **The trade-off:** The checkpoint occasionally flags a harmless umbrella as a weapon, causing delays and frustration. Calibrating the sensitivity to balance security and user experience is an ongoing challenge. The trade-off is false positives for defense in depth.

### Tiny Numeric Example

**Defense effectiveness on 1,000 adversarial prompts:**

```
No classifier:
  Jailbreaks succeed:   180 / 1,000 (18.0%)
  False negative rate:  18.0%

Keyword filter only:
  Jailbreaks succeed:    90 / 1,000 (9.0%)
  False positives:       45 / 1,000 (4.5%)

Constitutional classifier:
  Jailbreaks succeed:    12 / 1,000 (1.2%)
  False positives:        8 / 1,000 (0.8%)
```

**Performance on benign prompts:**
```
Keyword filter accuracy:   91.5%
Constitutional classifier:  97.8%
```

### Common Confusion

1. **"A constitutional classifier is the same as the model's safety training."** The main model is trained to refuse harmful requests. The classifier is a separate model trained to detect them. They are complementary layers.

2. **"It only catches explicit harmful words."** A well-trained classifier detects semantic intent, obfuscation, roleplay, and encoding tricks that keyword filters miss.

3. **"The classifier must be larger than the main model."** It is usually much smaller. Input classification is cheaper than generation because it is a single forward pass, not autoregressive decoding.

4. **"Once trained, it never needs updating."** Jailbreaks evolve. The classifier must be retrained periodically on new adversarial examples discovered by automated red-teaming.

5. **"Constitutional classifiers eliminate all false positives."** No filter is perfect. Calibration is required to avoid blocking benign but edgy content such as medical or legal discussions.

6. **"They are only useful for text."** The same principle applies to image inputs, code submissions, and multi-modal prompts. A vision classifier can detect harmful imagery before it reaches a multimodal model.

7. **"Constitutional classifiers replace the need for safe training data."** They add a layer on top of safe training. Both are necessary: safe training reduces harmful capabilities, and classifiers catch attempts to elicit them.
